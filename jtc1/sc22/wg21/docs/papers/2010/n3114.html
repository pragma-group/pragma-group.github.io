throw() becomes noexcept

During the Rapperswil meeting the Library Working Group decided to change the empty dynamic exception specifications in the librrary to the noexcept specification in the standard library. There was unanimous consent about this change. This paper presents proposed wording for this set of changes. The paper addresses National Body comments CH 16 and GB 60.

Identified changes

This paper proposes the following changes:
  • Delete every empty dynamic exception specification in the library.
  • Add noexcept specifications to the places where formerly there was an empty exception specification.
  • Delete every empty dynamic exception specification from destructors.
The exception specifications for class auto_ptr have not been updated as it is a deprecated class.

Acknowledgments

I would like to thank Alisdair Meredith for guiding me in the writing of this paper and providing me with hints to improve it.

Chapter 17: Library Introduction

In 17.6.3.6 replacement functions for new and delete with a nothrow_t argument had no exception specification. For consistency with changes performed in Chapter 18, this list has been updated to use noexcept.
